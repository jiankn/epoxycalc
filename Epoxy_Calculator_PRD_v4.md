# 环氧树脂项目规划器 (Epoxy Project Planner) - V4 PRD

## 0. 文档定位
本文件为当前最新版 PRD。

V4 不再采用 “MVP -> 再扩张” 的思路，而是按 **一次性全量上线** 的方式设计：

- 一次性完成全站信息架构
- 一次性覆盖第一层、第二层、第三层关键词
- 一次性做好工具、内容、内部链接、技术 SEO、AdSense / Affiliate 承接位

但这里有一个关键前提：

**一次性把站做完整，不等于一次性就能拿到排名。**

本项目可以做到低维护，但不能把排名建立在“只上线一次、后面几乎不动”这个假设上。SEO 的正确目标是：

- 上线时就把结构做对
- 让单个页面具备足够高的任务完成度
- 尽量减少后续返工
- 后续维护尽量只剩：外链、少量商家更新、Search Console 调整、少量内容补强

## 1. 项目目标
打造一个围绕 `epoxy calculator / epoxy resin calculator / river table epoxy calculator / deep pour epoxy calculator / coverage / cost / converter` 的高信任工具站，在上线第一天就具备完整的关键词覆盖和完整的转化链路。

## 2. 核心判断

### 2.1 可以成立的部分
- 用户需求真实存在。
- 目标关键词背后有明确任务意图和商业价值。
- 工具型页面天然适合承接高意图搜索。
- 用 AI 一次性完成全站开发是可行的。

### 2.2 不能自欺的部分
- Google 没有承诺新站 1-2 个月内给核心词首页排名。
- 一次性铺很多页面，如果内容和功能高度重复，容易踩到 doorway / scaled content abuse 风险。
- 头部关键词已经有品牌站、工具站、内容站和电商站在竞争。
- 真正决定排名的不是“上线页面数”，而是页面是否比现有 SERP 结果更完整地完成任务。

### 2.3 本项目的最优打法
不是 “一个关键词一个薄页面”，而是：

- **一个页面吃一个意图簇**
- **一个页面同时满足工具 + 解释 + 决策 + 转化**
- **全站一次性上线，但只索引真正值得索引的页面**

这是兼顾“全包一次性搞好”和“避免薄页风险”的更优解。

### 2.4 页面定义原则
本项目后续所有页面设计都必须遵守以下规则：

**一个页面，一个核心搜索意图，一个主关键词簇。**

说明：
- 一个页面可以覆盖多个高度相关的关键词变体
- 但这些关键词必须共享同一个搜索意图
- 如果关键词背后的任务、输入项、结果形式明显不同，才应该拆成新页面

这条原则高于“一个页面一个 exact keyword”的旧式做法。

## 3. 排名现实判断

### 3.1 1-2 个月内的现实预期
基于当前 SERP 和 Google 官方关于抓取/索引的说明，1-2 个月内更现实的结果是：

- 页面被抓取和索引
- 部分长尾和支持型词拿到 impressions
- 极少数竞争弱的变体词可能进前 10-20

不现实的保守承诺：
- `epoxy calculator`
- `epoxy resin calculator`
- `epoxy coverage calculator`

在 1-2 个月内稳定进入 Google 首页。

### 3.2 更可行的目标
上线时就把能覆盖的意图全部覆盖好，但在预期管理上区分：

- **第一层关键词**：先争取被理解、收录、拿初始相关性
- **第二层关键词**：争取进入前 20-30
- **第三层关键词**：争取更快拿到前 10-20，甚至部分首页

## 5. 全站 SEO 结构策略

### 5.1 结构原则
一次性全站上线，但分为两类 URL：

#### A. 可索引的核心页面
这些页面要独立做强，承担关键词排名。

#### B. 不索引的工具状态页 / 参数页 / 过滤页
这些页面服务用户体验，但不参与索引，避免重复内容和索引膨胀。

这比“所有状态都索引”更有效，也更安全。

### 5.2 为什么不能给每个词都独立造页
因为对于 Google 来说：

- 太多相似页面会像 doorway
- 太多轻微变体会像 scaled content
- 太多低差异 conversion pages 会稀释站点质量感知

所以更优方案是：

- **头部词 -> hub 页面**
- **场景词 -> scenario calculator 页面**
- **决策词 -> guide 页面**
- **单位词 -> converter hub 页面**

## 7B. 拆页判断规则

### 7B.1 必须拆页的情况
- 搜索意图明显不同
- 输入参数明显不同
- 用户要完成的任务不同
- 结果输出结构不同
- 当前 SERP 已明显分化为不同类型结果

### 7B.2 不应拆页的情况
- 只是词序变化
- 只是近义词替换
- 只是单位变化
- 只是 `epoxy` / `resin` 的自然互换
- 只是一个词簇内的长尾变体

### 7B.3 典型例子
应该合并到同一页：
- `epoxy calculator`
- `epoxy resin calculator`
- `how much epoxy do i need`

应该拆成不同页：
- `river table epoxy calculator`
- `garage floor epoxy calculator`
- `epoxy unit converter`
- `deep pour vs table top epoxy`

## 9. 更有效的排名打法

### 9.1 不是 pSEO，而是 Intent Cluster SEO
核心思想：

- 用更少但更强的页面覆盖更多关键词
- 每页覆盖一组真实相关的词
- 页面之间有清晰的层级与链接关系

### 9.2 SERP 混合意图打法
当前目标 SERP 往往混合出现：

- calculator
- brand page
- guide article
- help center

所以最佳页面不是纯工具，也不是纯文章，而是：

**工具 + 解释 + 教程 + 决策**

### 9.3 页面必须具备的独特点
每个核心页面都至少有以下独特点：

- 透明公式
- 原创测量说明
- 结果解释
- waste / seepage / seal coat / edges
- Compare scenarios
- cost estimation
- FAQ

如果只是“输入尺寸 -> 输出数字”，很难打过现有结果。

## 10. 内链策略

### 10.1 站内层级
- 首页链接所有核心 hub
- hub 页面链接所有 scenario 页面
- scenario 页面回链相关 guide
- guide 页面嵌入 calculator 并回链 scenario 页面
- converter 页面回链主 calculator 页面

### 10.2 Anchor text 原则
链接文本要明确描述目标页内容，例如：

- `river table epoxy calculator`
- `deep pour epoxy calculator`
- `how to measure a river table for epoxy`

不要堆砌无意义的 “click here”。

## 11. 技术 SEO 方案

### 11.1 页面输出方式
- SSR / SSG 优先
- 首屏关键文字必须直接存在于 HTML 中
- 不依赖客户端执行后才出现核心文本

### 11.2 索引控制
- 参数页、计算结果状态页、筛选页：`noindex`
- 可分享的状态链接：可保留，但 canonical 指向主页面
- 避免 query 参数页进入索引

### 11.3 Sitemap
一次性准备：
- `sitemap.xml`
- `sitemap-calculators.xml`
- `sitemap-guides.xml`
- `sitemap-site.xml`
- `sitemap-index.xml`

### 11.4 Structured Data
建议实现：
- `BreadcrumbList`
- `Article` 用于 guide 页面
- `Product` 仅在确实合适的产品信息区使用

说明：
- Google 没有专门的 calculator rich result 类型
- 结构化数据的目标是帮助理解，不是幻想特殊排名捷径

### 11.5 Search Console
上线当天完成：
- 验证站点
- 提交 sitemap index
- 检查抓取与索引
- 观察 title / snippet 改写情况

## 12. 信任与 E-E-A-T

### 12.1 站点必须有的信任页
- About
- Authors
- Methodology
- Contact

### 12.2 Methodology 页面必须说明
- 公式来源
- 单位换算常数
- waste / seepage / seal coat 的加入逻辑
- 结果为何只是 estimate
- 何时应该联系树脂品牌确认

### 12.3 作者信息
不能匿名堆内容。

至少需要：
- 作者姓名
- 简介
- 与木工 / 树脂 /工程计算相关的背景
- 内容更新时间

## 13. UI / UX 原则

### 13.1 页面气质
- Industrial Minimalist
- Mobile-first
- Tool-first
- Trust-first
- Not e-commerce-first

### 13.2 核心要求
- 高对比浅色主题
- 单手操作优先
- 48px 以上触控目标
- 大号数字输入框
- 实时计算但稳定不抖动
- 结果卡清晰醒目，但不做过重黑卡

### 13.3 结果后的自然停留
必须靠这些模块提升有效停留：

- Why this estimate changed
- Compare scenarios
- How to measure
- Common mistakes
- Project checklist

不是靠隐藏结果或强制弹窗。

## 14. 商业模式

### 14.1 基本逻辑
工具优先，Affiliate 其次。

### 14.2 CTA 顺序
必须遵守：

1. 结果
2. 结果解释
3. 产品类型建议
4. 商业 CTA

### 14.3 CTA 文案
优先：
- `View Deep Pour Options`
- `See Recommended Kit Sizes`
- `Check Price on Amazon`

避免默认使用：
- `Save 10%`
- `Best Deal`
- `Buy Now`

### 14.4 AdSense
可加，但必须克制：
- 不挡内容
- 不破坏 calculator 使用
- 不压过结果区

## 15. 一次性上线的内容标准

### 15.1 每个可索引页面都必须有
- 独立标题
- 独立 H1
- 独立开头摘要
- 实际可用工具或实际有价值内容
- FAQ
- 站内推荐链接

### 15.2 不能接受的页面
- 只是换个关键词的复制页
- 只有工具没有解释
- 只有文章没有任务完成能力
- 只有 Affiliate 卡片没有独立价值

## 16. 上线清单
- 全站 URL 结构确定
- 所有可索引页面完成
- 所有 noindex / canonical 策略完成
- 核心计算逻辑测试完成
- 内链完成
- Breadcrumb 完成
- Sitemap index 完成
- About / Authors / Methodology 完成
- Search Console 配置完成
- AdSense 位预留但不影响体验
- Affiliate 位已做但不过度营销

## 17. 最终决策标准
这个项目不是靠“页面数量”赢，而是靠以下五点：

1. 是否一次性把所有核心意图覆盖完整
2. 是否避免了 doorway / 薄页 / 索引膨胀
3. 是否每个关键页面都比现有 SERP 更完整
4. 是否让头词、中词、长尾词在同一架构里彼此输血
5. 是否即使后续维护很轻，也能保持站点结构长期有效

如果这五点成立，这个项目适合一次性全量建设；如果不成立，只是一次性造很多页面，并不会更容易拿排名。
